WWW is collection of HTML (Hypertext Markup Language).
HTML is the main doc type on the web. Anything can exist on the web.

  • Text: what you see
  • Markups: what it looks like
  • Reference to other docs: e.g. Images or Video
  • Links to other pages

Elements: refer to everything within the open/close tags <openingtag> "content" </closingtag>

Tags: When you forget to close tags everything after tag is updated/changed until you add closing tag.

Whitespace: text on multiple lines. Single line unless you add break tag

Container Element: do nothing but contains stuff inside them. You can add style to your containers using CSS.

HTML Attributes: Tags can have multiple attributes

You don`t have to memorize each tag. Programers don`t - Google is your friend!

HTMLTag List

<tag attribute = "value">content </tag>

  • Anchor tags <a> are for making links to < attribute name = "value"> content </a<
    • <attribute name = "value"> content </tag>: for example: Pinterest
  • Image tags <img> are for adding images
    • Alt text is required for broken request or the blind
    • Have no close tags. They are known as VOID tags
    • <img src = "URL"> alt ="text": for example
  • <br> tag void tag/no close. Starts new line
  • <p> tag paragraph; needs a close tag.
    • <p> Content </p>
  • Emphasis (italics): <em> </em>
  • Bold: <b> </b>

Inline vs Block:

  • Inline: <br >; <em >; <img >; <b >; <a >; <strong >
  • Block: <p >; <div >; <form >
    • Block tags create an invisible box

Span vs Div

  • Span is inline: <span> text </span>
    • With CSS: <span class = name > text </span>
  • Div is block: <div> text </div>
    • With CSS: <div class = "name"> text </div>

Lesson 2: Structured Docs with HTML

Terms:

HTML - HyperText Markup Language - the standard markup language used to create web pages.

CSS - Cascading Style Sheets - style sheet language used for describing the look and formatting of a document written in a markup language.

DOM - Document Object Model - a cross-platform and language-independent convention for representing and interacting with objects in HTML (and other markup languages). The nodes of every document are organized in a tree structure, called the DOM tree.

How do you translate a visual design into website?

Think about it as a house:

  • HTML: structure is the walls
  • CSS: style of house: color of walls
  • Javascript: interactive components: garage opener

The browser builds a tree like structure of the web page. Programmers refer to this as DOM

HTML and CSS are both languages. HTML controls the structure (syntax + rules) and CSS controls the style (syntax + rules). Basic word in HTML is a tag and the browser turns into elements that forms trees (knows because of the DOM). Each tag has a name or type and each tag can have an attribute.

All website are a compilation of boxes. We define boxes with <div> tags. Taking the design mock and breaking down the design into boxes or visual element.

Class attributes are like labels on boxes. They are there to remind you what is inside of them.

Lesson 3: CSS Style/HTML Structure

CSS - Cascading Style Sheets

  • You can think about CSS as a search and replace tool
  • Order matters: it matters where you define the rules and in what order they are applied:
    • The default style of a browser (different browsers have slightly different styles)
    • Stylesheet in a separate file (this is what you will be mostly using)
    • Stylesheet inside HTML (this can be done for small projects but is not ideal)
    • Inline style in an element (this can also be done but should be avoided)
  • Specifics matter: rules are applied not only to the elements they directly match, but also to all of those elements' child elements. More specific rule takes effect
  • Inheritance prevents certain properties from being declared over and over again in a style sheet
  • Whitespace between properties and selectors is ignored
  • The way to format CSS for readability is to indent
  • Everything on the webpage is a box. HTML elements are boxes and each box has 4 components. Hard to get the size of a box just right. Use percent`s rather than pixels. Use and set box sizing attribute to border box
  • Do not forget-- Divs are block elements (as opposed to inline), so by default they take up the entire width of a page. Adding rule display: flex; will override let divs appear next to each other
  • Adding images: make folder and review naming convention for characters. Do not forget alt attribute

Code, Test and Refine:

  • Look for natural boxes
  • Look for repeated style and semantic elements
  • Write HTML
  • Apply styles (largest to smallest)
  • Fix small things
  • Test within DevTools (allows you to make changes without have to change code)
    • Sources tab to see code.
  • Compare your page to design. Does it resemble mock-up?
